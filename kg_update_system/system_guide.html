详细系统功能清单

这份清单用于快速判断系统能完成哪些工作。左侧是从资料到研究输出的主流程，右侧可展开查看每类功能的具体能力、输入输出和使用价值。

主流程总览

01
接收资料术语、文献、指南、病历、PDF、Word、图片、Markdown、表格。
02
抽取知识识别概念实体，提取关系、证据句、来源文件和候选新实体。
03
生成图谱输出小图谱、主图谱、更新图谱和树状结构，支持点击与筛选。
04
多轮智能问答基于图谱检索、PubMed 提取资料、大模型和本机问答记忆形成连续回答。
05
人工核查与合并查看小图谱、CSV 和报告后，用“接受新增候选实体”开关决定是否写入更新图谱。
06
结果文件沉淀生成 CSV、GraphML、JSON、HTML、Markdown；填写 Obsidian Vault 时额外写入笔记文件。
1. 多来源资料输入

把分散资料统一送入知识图谱系统，减少手工复制、整理和重新命名的负担。

  • 支持粘贴文本、填写文件路径、填写目录路径和 GUI 拖拽上传。
  • 支持 NCIt 术语、PubMed 摘要、指南段落、病历摘要、研究笔记。
  • 支持 `.txt`、`.md`、`.csv`、`.json`、`.html`、Word、PDF、图片等资料。
  • 对 PDF、图片等读取失败情况生成解析诊断，提示是否需要 OCR 或大模型视觉读取。
2. 概念与实体识别

系统以主知识图谱为词典核心，识别结直肠癌领域中的概念、实体和同义表达。

  • 识别主图谱已有实体，标记为 `existing`。
  • 发现规则抽取的新候选实体，标记为 `new_candidate_rule`。
  • 调用大模型时补充新候选实体，标记为 `new_candidate_llm`。
  • 保留实体类别、定义、同义词、来源资料和证据句。
3. 关系抽取与证据记录

把实体之间的连接从文本中提取出来，使资料不再只是段落，而能变成可导航的知识网络。

  • 继承主图谱中已有关系，辅助构建新资料局部网络。
  • 根据同文共现生成候选关系，适合快速发现关联线索。
  • 大模型增强时可抽取更细的实体关系和简短证据。
  • 关系表保留 `source_docs` 和 `evidence`，便于回到原始资料核对。
4. 小知识图谱生成

每批新资料会先形成局部小图谱，作为进入主图谱前的核查入口；系统提供候选状态和证据，但最终判断仍由人工完成。

  • 生成 `mini_nodes.csv` 和 `mini_edges.csv`，便于表格审核。
  • 生成 `mini_browser.html`，用于浏览局部实体关系网络。
  • 生成 `mini_tree.html`，用于按树状结构查看新资料知识。
  • 生成 `mini_report.json`，记录抽取数量、解析问题和模型调用问题。
5. 主知识图谱与树状结构浏览

系统提供网络图和树状图两种视角：网络图看关系，树状图看层级。

  • 主知识图谱展示结直肠癌领域实体之间的网络结构。
  • 树状结构展示概念上下位关系，适合做领域框架梳理。
  • 支持按疾病、基因、解剖部位、治疗、标志物等类别筛选。
  • 点击节点可查看类别、定义、同义词、状态、证据和邻接关系。
6. 多轮智能问答 GraphRAG

问答不是只让模型自由发挥，而是先从图谱检索相关实体和关系，再结合最近对话记忆组织上下文生成回答。

  • 支持基于主知识图谱检索相关节点和边。
  • 支持调用 DeepSeek 生成“综合 AI 回答”。
  • 支持结合 PubMed 提取文件、新资料摘要和图谱证据回答。
  • 可区分本地图谱依据与第三方资料或模型背景知识。
  • 支持多轮追问，例如“它和治疗有什么关系”“继续解释上一个概念”。
  • 问答记忆保存在本机浏览器，可设置记忆轮数，并用“清空记忆”或“导出问答”管理记录。
7. 大模型增强处理

大模型用于补足规则抽取的不足，尤其适合处理复杂中文医学文本、图像资料和关系表达。

  • 可对新资料生成简要总结，并在 Markdown 中保存更完整分析。
  • 可补充规则难以识别的新实体、别名、定义和关系。
  • 可读取部分图片或扫描 PDF 内容，但需要人工核对。
  • 可关闭大模型增强以提高速度，只保留规则抽取和基础摘要流程。
8. 人工核查、合并更新与版本保留

合并功能负责把小图谱写成一个“更新版图谱目录”。它不是自动审核平台，也不会直接覆盖原始主图谱。

  • 合并前主要通过 `mini_nodes.csv`、`mini_edges.csv`、小图谱 HTML 和 `mini_report.json` 人工核查。
  • 不勾选“接受新增候选实体”时，候选新实体不会进入更新图谱，只更新已有实体证据和可保留关系。
  • 勾选“接受新增候选实体”后，候选节点会写入 `runs/gui_updated_kg` 这个更新目录。
  • 系统会生成 `merge_report.json` 和 `_backup_source_时间戳` 备份目录，记录合并参数和来源。
9. Markdown 笔记与 Obsidian 文件联动

系统会生成 Markdown 总结文件；如果在 GUI 中填写 Obsidian Vault 路径，会把同类笔记写入指定库和子目录。它目前不是 Obsidian 插件，也不会自动为每个实体建立双链卡片。

  • 新资料抽取后固定生成 `runs/.../mini_note.md`，记录摘要、解析备注、文档、节点和关系。
  • 合并更新后生成 `runs/.../kg_note.md`，记录更新版图谱的节点和关系摘要。
  • 填写 Vault 路径时，系统会在指定子目录写入 `.md` 笔记，并可生成 `obsidian://open` 打开链接。
  • 后续双链、实体卡片、标签体系和文献笔记结构，需要在 Obsidian 中人工继续整理。
10. 数据文件导出与外部工具衔接

系统会生成多种本地文件格式，便于手动导入外部工具。当前不是自动调用 Gephi、Cytoscape、Neo4j 或统计软件的外部分析流水线。

  • 小图谱导出 `mini_nodes.csv`、`mini_edges.csv`、`mini_graph.graphml`、`mini_cytoscape.json`、`mini_browser.html`。
  • 更新图谱导出 `kg_nodes.csv`、`kg_edges.csv`、`kg_graph.graphml`、`kg_cytoscape.json`、`kg_browser.html`。
  • CSV 可用于 Excel、Python 或 R 的后续人工分析；GraphML 可手动导入 Gephi、yEd 或 NetworkX。
  • Cytoscape JSON 可作为 Cytoscape 生态的数据输入；HTML 主要用于本地浏览和展示。
11. 审核辅助与质量提示

当前系统提供审核辅助信息，但尚不是带审批流、权限和规则库的正式质控平台。

  • 通过 `kg_status` 区分已有实体、新候选实体和模型候选实体。
  • 通过 `evidence` 与 `source_docs` 回到原文核查。
  • 通过 `parse_errors` 和 `llm_errors` 发现资料读取或模型抽取问题。
  • 通过合并前人工勾选机制，避免候选实体未经确认进入更新图谱。

      常见问题

      这里保留最常见的故障判断入口，便于学生或第一次使用者快速定位。

      GUI 打不开

      先运行 `python .\kg_update_system\kg_gui.py`，再访问 `http://127.0.0.1:8765`。如果仍打不开，检查 PowerShell 是否停在报错处，或端口 8765 是否被旧进程占用。

      PDF 或图片未读取到资料

      优先检查文件是否是扫描件、图片是否清晰、PDF 是否可复制文本。扫描件需要 OCR；含复杂表格或低清图片时，大模型总结也可能缺少证据句。

      DeepSeek 调用慢

      减少一次投入的资料量，先不开启“大模型增强”跑规则抽取；需要补关系和摘要时再开启模型。长 PDF 建议拆成章节处理。

      图谱太乱

      优先使用类别筛选和 top-k 检索。top-k 表示“最多取前 k 个最相关结果”，数值越大覆盖越多，但页面和答案会更杂。

      新实体要不要合并

      先看候选实体是否真属于结直肠癌领域、是否与已有同义词重复、是否有来源证据。确认后再勾选“接受新增候选实体”。

      第三方资料如何标注

      本地图谱来自系统已有 CSV/HTML/GraphML；网络或模型外部知识应在回答里单独标为“第三方资料/模型背景知识”，避免和本地证据混淆。

      本页为本地静态说明书，可直接双击打开；实际分析、抽取、问答和合并请在 GUI 工作台中执行。
