# 结直肠癌知识图谱 GUI 工作台

## 1. 启动

在 PowerShell 进入工作目录：

```powershell
cd <项目根目录>
```

如果要调用 DeepSeek，先设置 API Key：

```powershell
$env:KG_LLM_API_KEY="你的DeepSeek API Key"
```

启动 GUI：

```powershell
python .\kg_update_system\kg_gui.py
```

浏览器会自动打开：

```text
http://127.0.0.1:8765
```

如果没有自动打开，手动复制这个地址到浏览器。

## 2. GUI 对应整体架构

### 2.1 数据输入层

界面位置：`新资料抽取`

支持两种输入：

- 直接粘贴文本：PubMed 摘要、指南段落、病历摘要、研究笔记。
- 填写本机文件/目录路径：支持 `.txt`、`.md`、`.csv`、`.json`、`.html`。

### 2.2 知识抽取层

界面位置：`新资料抽取`

当前实现：

- NER 实体识别：主图谱词典匹配 + 正则候选实体抽取。
- 关系抽取：已有图谱关系继承 + 同文共现关系 + 可选大模型关系抽取。
- 事件抽取：预留在大模型抽取层，可扩展到治疗响应、突变检测、转移事件。
- 属性补全：实体类别、来源资料、证据句、别名、定义。

输出小图谱：

- `mini_nodes.csv`
- `mini_edges.csv`
- `mini_browser.html`
- `mini_graph.graphml`
- `mini_cytoscape.json`

### 2.3 知识存储层

界面位置：底层由脚本自动维护。

当前可执行实现：

- CSV：主数据表，最便于人工审核。
- GraphML：Gephi / yEd / NetworkX。
- Cytoscape JSON：Cytoscape 系列工具。
- HTML：本地可视化浏览。

架构预留：

- Neo4j：适合正式图数据库部署。
- Chroma：适合文献段落、病历片段、指南文本的向量检索。
- SQLite：适合保存资料来源、抽取任务、审核状态、版本号。

当前 GUI 先用轻量本地文件系统跑通闭环，后续可把 `kg_incremental.py` 的读写函数替换为 Neo4j / Chroma / SQLite 适配器。

### 2.4 应用层

界面位置：

- `智能问答`：GraphRAG 问答。
- `图谱浏览`：打开主图谱、小图谱、更新图谱。
- `合并更新`：把新资料小图谱合并进主图谱。

## 3. 智能问答界面

输入问题后，系统执行：

```text
问题 → 图谱检索相关节点/边 → 组织上下文 → 调用 DeepSeek → 返回答案 + 图谱依据
```

界面会显示：

- 回答正文。
- 相关概念/实体。
- 相关关系。
- 原始 JSON，便于调试和复用。

如果没设置 `KG_LLM_API_KEY`，系统不会调用模型，只返回图谱检索上下文。

## 4. 新资料抽取界面

推荐流程：

1. 粘贴一段新资料，或填写资料目录。
2. 先选择“不用模型，先规则抽取”。
3. 查看生成的 `mini_browser.html` 和 `mini_nodes.csv`。
4. 如果规则抽取不够，再选择“调用 DeepSeek 增强”。
5. 人工审核新增候选实体。

## 5. 合并更新界面

合并前重点检查：

```text
mini_nodes.csv 中 kg_status 不是 existing 的行
```

如果确认新增实体可进入主图谱，勾选：

```text
接受新增候选实体并写入更新图谱
```

合并不会覆盖原始主图谱，而是生成新的输出目录，并自动备份原始图谱。

## 6. HTML 是否能显示问答结果

可以。当前 GUI 已经在浏览器 HTML 中显示：

- 提问输入框。
- 模型/检索选项。
- 结构化答案。
- 相关概念/实体卡片。
- 相关关系卡片。
- 原始 JSON。

也就是说，问答不再只在命令行显示。

## 7. 下一步可扩展项

### 7.1 接 Neo4j

把 `kg_nodes.csv` 与 `kg_edges_research_core.csv` 导入 Neo4j 后，GUI 的问答检索层可以从 CSV 检索替换成 Cypher 检索。

### 7.2 接 Chroma

新资料进入系统时，把原文切块写入 Chroma。问答时同时检索：

```text
图谱邻域 + 文本向量片段
```

这样能回答更接近“原文证据”的问题。

### 7.3 接 SQLite

SQLite 保存：

- 资料文件清单。
- 每次抽取任务。
- 新增候选实体审核状态。
- 图谱版本号。
- API 调用日志摘要。

这会让系统从“脚本工具”升级为“可管理知识库”。
