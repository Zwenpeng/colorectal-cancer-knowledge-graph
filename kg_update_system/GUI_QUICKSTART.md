# GUI 快速启动与图谱浏览说明

## 启动

```powershell
cd <项目根目录>
python .\kg_update_system\kg_gui.py
```

打开：

```text
http://127.0.0.1:8765
```

## DeepSeek 配置

当前程序支持三种写法：

1. 推荐：环境变量 `KG_LLM_API_KEY`
2. 配置文件字段 `api_key`
3. 兼容写法：真实 key 写在 `api_key_env`

你已经把接口信息写入 `config.deepseek.json`，GUI 会自动检测。

## 图谱浏览

新版不再用 `file://` 打开 HTML，而是在 GUI 中内嵌显示：

- 主知识图谱
- 最近小图谱
- 最近更新图谱
- 自定义 HTML 路径

这样可以避开中文路径、本地文件权限、默认浏览器未响应等问题。

## 常见操作

- `智能问答`：输入问题，返回答案、相关实体、相关关系。
- `新资料抽取`：粘贴资料或填写目录，生成小图谱。
- `合并更新`：把小图谱合并为更新版主图谱。
- `图谱浏览`：直接在页面中查看图谱。

合并后的更新版图谱现在使用与主知识图谱相同的浏览标准：导览视图、分子/解剖/临床/治疗视角、节点类型筛选、搜索和右侧详情面板。

## 学生版使用顺序

1. 打开 GUI 后先看顶部状态：显示 `AI 可用` 即可直接问答。
2. 进入 `智能问答`，点 `测试 AI 连接`，看到 `DeepSeek连接正常` 后再提问。
3. 不知道目录怎么填时，进入 `设置`，点击路径卡片：
   - 主知识图谱
   - DeepSeek 配置
   - 默认小图谱输出
   - 默认更新图谱输出
4. 进入 `图谱浏览`，点 `主知识图谱`，图谱会直接嵌入页面显示。
5. 有新资料时，进入 `新资料抽取`，先粘贴文本，再点 `生成小图谱`。
6. 检查小图谱后，进入 `合并更新`，再决定是否勾选 `接受新增候选实体`。

## DeepSeek 增强抽取为什么慢

`新资料抽取` 里有两种方式：

- `不用模型，先规则抽取`：速度快，适合先粗筛。
- `调用 DeepSeek 增强`：会让模型读资料并抽实体/关系，通常需要 1-3 分钟。

推荐流程：

1. 先用规则抽取生成小图谱。
2. 如果新增实体不够，再切换到 DeepSeek 增强。
3. 增强抽取时不要刷新页面。
4. 结果里看 `LLM实体`、`LLM关系`、`DeepSeek 部分片段抽取失败`。

如果 DeepSeek 某个片段失败，系统不会中断，会保留规则抽取结果，并在结果里显示失败片段数。

## AI 问答是否真的调用模型

问答结果标题旁会显示：

- `AI 已回答`：已经调用 DeepSeek。
- `仅图谱检索`：没有调用模型，只展示图谱检索结果。

如果想强制调用模型，`智能问答` 页面中的 `是否调用模型` 选择 `调用 DeepSeek`。

## 图谱参考数量是什么意思

界面里的“图谱参考数量”就是原来的 `top-k`。

含义：系统先从知识图谱里找和问题最相关的概念/实体，再交给 AI。这个数字表示最多找多少个节点。

- 数字小：更快，依据更集中。
- 数字大：覆盖更广，但回答会慢一点，也可能引入不太相关的节点。

推荐：

- 普通问题：`20`
- 很具体的问题：`10`
- 综述型问题：`30-50`

## 第三方资料模式

在 `智能问答` 中，`第三方资料` 可以选择：

- `不联网，只用图谱`
- `检索 PubMed，并标注第三方来源`

启用 PubMed 后，答案会分成：

- 图谱依据：来自本地知识图谱。
- 第三方资料依据：来自 PubMed 文献，会显示 PMID、标题、期刊、年份和链接。

注意：第三方资料不是主图谱的一部分，除非你后续把它抽取成小图谱并合并。

## 三种回答方式

`智能问答` 里新增了“回答方式”：

1. `图谱优先`
   - 主要依据本地知识图谱。
   - 适合问：某个实体和结直肠癌有什么关系、某个概念有哪些下位概念。

2. `PubMed文献综述`
   - 自动检索 PubMed。
   - AI 会总结检索到的文献摘要，并列出 PMID/URL。
   - 适合问：某治疗、突变、biomarker 的研究进展。

3. `综合AI回答`
   - 同时使用本地图谱、PubMed 文献、AI 已学医学知识。
   - 回答会分开标注：
     - 图谱依据
     - PubMed 第三方资料依据
     - AI 综合医学知识
   - 适合像正常 AI 一样问开放问题，但仍保留来源边界。
