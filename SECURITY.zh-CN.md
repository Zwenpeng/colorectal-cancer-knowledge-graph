# 安全策略

## 密钥与敏感资料

不得提交 API 密钥、含凭据的模型配置、浏览器历史、患者资料或未脱敏的临床笔记。

仓库已忽略 `kg_update_system/config.deepseek.json` 等本地配置文件。请从 `kg_update_system/config.example.json` 创建本地配置，并通过环境变量 `KG_LLM_API_KEY` 提供密钥。

## 漏洞与敏感内容报告

若发现凭据、敏感文件或安全缺陷，请不要在公开 Issue 中粘贴敏感内容。应先通过 GitHub 私信联系仓库维护者。

## 医学资料边界

仓库内资料是术语数据和合成/示例研究材料。使用增量流程处理本地病历前，必须完成去标识化，并遵守所在机构的数据治理要求。
